# 设计指南

## 设计原则

好的 PDF 文档需要层次清晰、视觉精致、布局合理，你的设计必须达成两个目标：
1.  Clarity: 信息层级分明，读者能快速定位关键信息，长文阅读不疲劳
2.  Aesthetics: 符合现代排版美学，字号字距行距协调，配色克制

实现的三个手段：
- **层级**：通过标题大小、颜色深浅、字重粗细建立信息骨架。H1/H2/H3 视觉差距必须明显。不要让读者猜测"这是标题还是正文"
- **留白**：优先用间距和留白分组内容，而不是边框和分割线。密集的文字块 = 阅读阻力，适当的留白 = 呼吸感
- **克制**：装饰元素（色块、线条、图标）只在服务于内容组织时使用。去掉后页面没变差的装饰，就不该存在。强调色同理——出现在越少的地方越有力

## 配色与排版

所有配色使用 Tailwind 类名，不用 HEX 色号。Step 2 定义好配色后，所有页面的用色不能超出这套定义。允许同色系深浅变化（如 `stone-700` 和 `stone-300`），但不引入新色相——一旦出现规范外的颜色，整套文档的统一感就会被打破。

具体约束：
- 默认浅色模式，深色仅在用户明确要求时使用
- 避免"AI 味"的通用选择：蓝紫渐变、Inter/Roboto/Arial 等最常见字体
- 章标题的视觉权重明显大于节标题（大小、颜色、装饰元素都要拉开差距）
- 正文行高不低于 1.6，段间距用 `mb-5` 保证段落分隔清晰

## 页面类型设计

PDF 文档有三类特殊页面，各自有不同的设计要求：

### 封面页

文档的第一印象。核心任务：让读者一眼知道这是什么、谁写的。

- 标题大字突出，可以占页面 1/3 以上空间
- 布局通常靠上居中 + 大面积留白
- 可用的装饰手段：大色块背景、几何图形、线条、图标（FontAwesome）
- 必须包含的信息：文档标题、副标题（可选）、作者/机构、日期

### 目录页

封面后的第一页。核心任务：快速展示文档骨架，支持点击跳转。

- 列出所有章节标题，每个标题是内部超链接（`<a href="#chapter-id">`）
- 目录条目格式：章节编号 + 标题（+ 可选页码）
- 使用视觉元素引导（编号、缩进、图标、引导线）帮助读者快速扫描
- 标题层级通过缩进或字号区分（章 > 节）

### 正文页

核心内容区。设计原则：
- 章标题突出，通常包含大号装饰元素（数字、色块、线条）
- 图表、表格、引用块有明确的视觉区分（边框、背景色、缩进）
- 页脚页码统一，放在每页底部居中或外侧
- 章与章之间保证从新页起始，短章节可自然流动不必每章断页

### 封底页（可选）

结尾页，构图和配色与封面呼应，形成闭合感。不是每份文档都需要。

### 字号层级（硬性约束）

全文使用 Tailwind 类名控制字号，按中文正式文档标准逐级放大：

| 元素                   | Tailwind 类名           |
| ---------------------- | ----------------------- |
| 封面主标题             | `text-7xl`              |
| 封面副标题             | `text-lg`               |
| 目录标题               | `text-3xl`              |
| 目录条目               | `text-base`             |
| h1 章标题              | `text-3xl`              |
| h2 节标题              | `text-xl`               |
| h3 小节标题            | `text-lg`               |
| 正文 p                 | `text-base`             |
| 表格 / 引用 blockquote | `text-sm`               |
| 代码 pre               | `text-sm`               |
| 封底标题               | `text-5xl`              |
| 封底文案               | `text-lg` / `text-base` |

### 间距规则（硬性约束）

章与章之间的缝隙是节与节之间的 **2 倍**：

| 元素          | Tailwind 类名 |
| ------------- | ------------- |
| h1 章前间距   | `mt-16`       |
| h2 节前间距   | `mt-8`        |
| h3 小节前间距 | `mt-7`        |
| 段落间距      | `mb-5`        |

## 视觉素材

图表、图标、图片的共同原则：服务于信息传达，不是填充空间。

**ECharts 数据图表**：只在有充足具体的数据时使用（数字、表格、百分比）。定性描述或数据点太少不需要图表。图表配色从设计规范中取色，不用 ECharts 默认配色。图表中的所有数据必须来自前序步骤搜集的资料，禁止为了让图表"好看"而补充资料中不存在的数据点。

**生图工具**：适合抽象概念插画、风格化配图、不存在的假想场景。生成时在 prompt 中指定色调和风格，与设计规范一致。不要用生图生成真实事物（产品实拍、人物肖像、具体建筑）——生图做不好这些，配错图的负面影响大于不配图。

**图片使用**：适合具体人物、产品实拍、地标建筑、数据截图等必须还原真实外观的场景。将浏览器下载的图片或用户上传的图片放在 `assets/` 目录，用相对路径引用。

**不需要图片时不要硬塞**：用色块、几何图形、图标（FontAwesome）、大字排版构建视觉焦点。
