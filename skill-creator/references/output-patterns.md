# 输出模式

当技能需要产出一致、高质量的输出时，使用以下模式。

## 模板模式

为输出格式提供模板，根据需要选择约束程度。

**适用于严格要求场景（数据格式、固定报告结构等）：**

```markdown
## 报告结构

始终使用以下模板：

# [标题]

## 摘要
[核心发现和关键结论浓缩]

## 核心发现
- 发现 1，附带支撑数据
- 发现 2，附带支撑数据

## 建议
1. 具体可执行的建议
2. 具体可执行的建议
```

**适用于灵活指引场景（需要根据内容自适应时）：**

```markdown
## 报告结构

以下是推荐的默认结构，可根据实际内容灵活调整：

# [标题]

## 摘要
[概述]

## 核心发现
[根据发现的内容自行组织子章节]

## 建议
[根据具体场景调整]
```

## 示例模式

当输出质量依赖于具体示例时，提供输入/输出对：

```markdown
## 提交信息格式

按以下示例生成提交信息：

**示例 1：**
输入：添加了 JWT 用户认证
输出：feat(auth): 实现基于 JWT 的认证

**示例 2：**
输入：修复了报表中日期显示不正确的问题
输出：fix(reports): 修正时区转换中的日期格式化
```

示例能帮助 Agent 理解期望的风格和详细程度，比纯文字描述更清晰。
