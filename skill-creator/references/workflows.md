# 流程模式

## 顺序流程

处理复杂任务时，将操作拆分为清晰的顺序步骤。通常建议在 SKILL.md 文件开头，先清晰概述整体流程：

```markdown
研究报告生成涉及以下步骤：

1. 需求理解与研究规划
2. 深度信息收集（search + fetch）
3. 素材落盘到 workspace
4. 大纲规划（确认门）
5. 分章节撰写（可并行）
6. 整合交付
```

## 条件流程

针对存在分支逻辑的任务，引导 Agent 完成各个决策节点

```markdown
1. 判断当前状态：
   **用户只给了模糊主题？** → 从需求澄清开始
   **用户带着素材来？** → 跳过搜索，直接进入规划
   **用户已有产出物，想改某一部分？** → 直接进入修改模式

2. 创建流程：[步骤]
3. 修改流程：[步骤]
```

## 并行流程

当任务包含多个互不依赖的子任务时，可并行推进以提升效率：

```markdown
## 工作流

1. 分析需求，拆分为独立子任务
2. 并行执行以下子任务：
   - 子任务 A：收集行业数据
   - 子任务 B：收集竞品信息
   - 子任务 C：收集政策法规
3. 汇总各子任务结果，整合输出
```
