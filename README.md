# WorkBuddy 技能集合

基于 WorkBuddy AI 助手的一套**轻量、高性价比**的技能工具集，聚焦 "够用、好用、容易维护"。

---

## 设计思路

市面上同类工具往往追求大而全——动辄上百个模板文件、几十套主题、繁复的动画引擎。代价是维护成本高、学习曲线陡、定制困难。AI 大量精力花在"选哪个模板""模板怎么填""这个参数怎么调"上——本质是在驾驭工具系统，而不是在创造内容。

在 WorkBuddy 这类 AI 助手里，skill 的每个文件最终都会加载到上下文中消耗 token。一份 150+ 文件的模板库意味着每次使用都在烧钱，而本项目的核心文件仅约 10 个，**完成同样任务的 token 消耗远低于同类方案**。

因为这套技能走的是另一条路：

**不堆模板，靠设计规范。** 没有预制的百套模板库，而是通过 Visual Anchor + 60-30-10 配色法则 + Tailwind 硬约束，每次直接生成原生 HTML。简单说：不替你决定长什么样，而是让你每次都"知道怎么把它做好看"。

**把 AI 的算力还给内容本身。** 本项目的设计规范只有几个明确的约束条件，AI 无需遍历数百个模板文件就能直接开始工作，推理能力更集中地用在内容质量上，而不是在工具的迷宫里绕路。

**核心产出物就是最终交付物。** HTML 源文件本身就是可浏览的幻灯片/文档，不依赖专有编辑器。改内容就是改 HTML，改样式就是换 Tailwind class。

## 技能一览

### ppt-creator — PPT 演示文稿生成

| | ppt-creator | 同类方案 |
|:---|:---|:---|
| **模板库** | 0 个预制模板，靠设计规范即时生成 | 往往 30+ 模板、150+ 资源文件 |
| **动画** | 不支持（PPT 静态即可，动画在转换中丢失） | 支持复杂 CSS/Canvas 动画（但转 PPTX 后同样丢失） |
| **核心文件数** | ~10 个 | ~150+ 个 |
| **上下文消耗** | 按需加载，token 开销低 | 全部加载，一次消耗大量 token |
| **学习成本** | 会 Tailwind + HTML 即可 | 需理解完整的模板/主题/动画体系 |
| **产出物** | 真实 .pptx，可二次编辑 | 部分方案仅输出 HTML 网页 |

工作流程：内容确认 → 视觉规划 → 逐页生成 HTML → 合并转 PPTX。支持新增/删除/移动页面、全局改风格。

### pdf-creator — PDF 文档生成

类似的轻量思路：没有预制文档模板，每次按设计规范从零生成。核心处理好了 PDF 最容易出坑的地方——页码管理（封面目录无页码、正文从 1 开始）、分页控制（禁止空 div 分页符）、目录超链接。

### skill-creator — 技能创建框架

把创建 WorkBuddy 技能这件事本身工程化了：有脚手架（`init_skill.py`）、打包工具（`package_skill.py`）、校验器（`quick_validate.py`）。五阶段工作流 + 确认门机制 + 反模式清单，确保产出的技能质量一致。

---

## 核心原则

- **确认门机制**：每次关键步骤前停下来让用户确认，不是一股脑推到完才发现不对
- **基础参数硬约束**：字号层级、配色规则、间距标准在配套文档中硬编码，不每次从头商量
- **多轮编辑不崩**：新增/删除/移动页面有脚本自动重编号，全局改风格有 Tailwind class 批量替换策略，方便对不满意的地方进行快速调整

---

## 技术栈

- **HTML + Tailwind CSS + FontAwesome** — 内容与样式，所有配色用 Tailwind 类名，不用 HEX
- **Playwright** — HTML 渲染，PDF 生成
- **PptxGenJS** — PPTX 转换
- **Python 3** — 辅助脚本（脚手架、打包、校验）
- **Bash** — 文件重编号工具

---

## 快速开始

将技能目录放入 `~/.workbuddy/skills/` 即可使用：

```bash
cp -r ppt-creator ~/.workbuddy/skills/
```

然后在 WorkBuddy 对话中触发：
- "帮我做一个关于 Q2 复盘的分析报告 PPT，8 页"
- "把这篇文档转成 PDF"
- "帮我创建一个读取网站并生成摘要的技能"

## 目录结构

```
workbuddy-skills/
├── skill-creator/            # 技能创建框架
├── ppt-creator/              # PPT 生成工具
└── pdf-creator/              # PDF 生成工具
```

每个技能目录结构统一：
```
skill-name/
├── SKILL.md                  主指令
├── scripts/                  可执行脚本
└── references/               参考文档
```
