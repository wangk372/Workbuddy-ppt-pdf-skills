# 设计指南

## 设计原则

好的 PPT 设计需要层次清晰、视觉精致、布局合理，你的设计必须达成两个目标：
1.  Clarity: 信息层级分明，观众能在 10 秒内抓住核心观点
2.  Aesthetics: 符合现代设计美学，讲究对齐和微观细节、配色协调

实现的三个手段：
- **对比**：通过大小、颜色深浅、字重粗细建立主次。不要平铺直叙
- **留白**：优先用间距和留白分组内容，而不是边框和卡片。边框本身是视觉噪音，留白是免费的分隔。拥挤 = 信息过载，呼吸感 = 精心筛选
- **克制**：装饰元素（图形、色块、线条）只在服务于内容组织时使用。去掉后页面没变差的装饰，就不该存在。强调色同理——出现在越少的地方越有力

## 配色与排版

所有配色使用 Tailwind 类名，不用 HEX 色号。Step 2 定义好配色后，所有页面的用色不能超出这套定义。允许同色系深浅变化（如 `stone-700` 和 `stone-300`），但不引入新色相——一旦出现规范外的颜色，整套 PPT 的统一感就会被打破。

具体约束：
- 默认浅色模式，深色仅在用户明确要求时使用
- 避免"AI 味"的通用选择：蓝紫渐变、Inter/Roboto/Arial 等最常见字体
- 每页标题区位置保持一致——让观众知道"标题总在那个位置"，减少认知成本

封面，目录和结尾是仪式感最强的三页：
- **封面页**（slide_01）：突出标题，内部布局通常为左对齐或居中
- **目录页**（slide_02）：使用视觉导览，使用3～6个含图标、数字的卡片展示整体内容框架
- **结尾页**（最后一页）：构图和配色与封面呼应，形成闭合感

字号层级（硬性约束）：

| 层级 | Tailwind |
|------|----------|
| 封面主标题 | `text-8xl` |
| 结尾大字 | `text-6xl` |
| 章标题 | `text-5xl` |
| 节标题 | `text-4xl` |
| 正文 | `text-2xl` |
| 小字/标签 | `text-xl` |

## 视觉素材

图表、图标、图片的共同原则：服务于信息传达，不是填充空间。

**数据图表**：按图表复杂度选方案。

- **柱状图、条形图、进度条**：用纯 CSS 实现（div + width% + 颜色 + 圆角），零 JS 依赖。html2pptx 会把每个柱子和标签转成独立的 PPTX 形状，后续在 PPTX 里可以单独改色改字。示例：`<div class="w-[62%] h-8 bg-red-600 rounded-r"></div>`
- **饼图、雷达图、热力图等复杂图表**：CSS 画不了。用 ECharts 预先生成为 PNG 图片，放入 `slides/assets/` 目录，在 HTML 中用 `<img src="assets/chart.png">` 引用。生成命令：
  ```bash
  node scripts/echarts_to_png.js slides/assets/chart.png '{"option":{...echartOption},"width":1000,"height":500}'
  ```
  注意：不要在 slide HTML 中引入 ECharts CDN 动态渲染——html2pptx 转换引擎不支持 data URI，只有静态图片文件能正确嵌入 PPTX。
- 图表配色从设计规范中取色，所有数据必须来自前序步骤搜集的资料，禁止编造。

**生图工具**：适合抽象概念插画、风格化配图、不存在的假想场景。生成时在 prompt 中指定色调和风格，与设计规范一致。不要用生图生成真实事物（产品实拍、人物肖像、具体建筑）——生图做不好这些，配错图的负面影响大于不配图。

**图片使用**：适合具体人物、产品实拍、地标建筑、数据截图等必须还原真实外观的场景。需要图片时，将浏览器下载的图片或用户上传的图片统一放入 slides 目录下的 `assets/` 文件夹。

**不需要图片时不要硬塞**：用色块、几何图形、图标（FontAwesome）、大字排版构建视觉焦点。
