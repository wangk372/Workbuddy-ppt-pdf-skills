# 编辑操作规则

核心原则：只改用户要求的部分，其余保持原样。新增或修改的页面必须与整套 PPT 的设计规范一致——修改前先读取 2-3 个已有 slide 的完整 HTML 源码，提取当前设计规范。

## 新增/删除/移动页面

详见 SKILL.md 的"编辑流程"部分。涉及文件重编号时优先使用 `scripts/reorder_slides.sh`。

文件命名必须保持 `slide_XX.html` 格式（下划线+两位数字）。

## 单页内容修改

改标题、改文案、调整某个元素。只改用户提到的内容，不要顺便调整其他元素——即使你觉得其他地方也可以改进。用户没提到的地方如果被改了，会让他们困惑"我只让你改标题，为什么布局也变了"。

如果修改导致内容溢出，按降级策略处理（减间距→缩字号→精简内容），并告知用户做了哪些额外调整。

## 全局风格调整

用户要求修改整套 PPT 的视觉风格（换配色、换字体等）。先明确新的设计规范并输出供用户确认，确认后逐页修改。优先用 Tailwind class 替换（如 `text-blue-600` → `text-emerald-700`），减少遗漏。
